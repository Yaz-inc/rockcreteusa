# 00 - Project Memory

This folder is the Drive-side mirror of the Git repo's `docs/project-memory/`. Everything here is plain Markdown so any AI tool or human teammate can read it.

**Canonical copy:** `https://github.com/newmindsgroup/rockcreteusa-project-blueprint/tree/main/docs/project-memory/` (Git, kebab-case filenames).
**This copy:** Drive (SCREAMING_CASE filenames). Updated by mirroring from Git after any substantive change.

## Files

| File | What it is |
|---|---|
| `OVERVIEW.MD` | What Rockcrete USA Website Rebuild is, who it's for, scope, goals, constraints. |
| `STATUS.MD` | Where things stand right now. |
| `DECISIONS.MD` | Append-only log of meaningful decisions. |
| `SESSION_LOG.MD` | Reverse-chronological log of working sessions. |
| `CONTEXT_INDEX.MD` | Map of every notable file across the three surfaces. |
| `TEAM_STRUCTURE.MD` | Team composition, ownership, review routing. |
| `INSTRUCTIONS.MD` | Copy-pasteable AI-instructions block. |
| `FOLDER_STRUCTURE.MD` | Canonical top-level folder layout. |
| `NAMING_CONVENTIONS.MD` | Naming rules for Drive, Git, VPS. |
| `WRITING_CONVENTIONS.MD` | Prose conventions including the no-em-dash rule. |

## How to use

Read these at the start of any working session before assuming you know the current state. Update them before closing out a working session. Mirror back to Git on every substantive change.

## Why two copies (Git + Drive)?

- **Git** is canonical, version-controlled, scannable from any IDE or AI tool.
- **Drive** is human-readable and shareable with non-technical teammates who don't use Git.
- The mirror prevents Drive-only teammates from being out of date.

If the two copies disagree, Git wins. Re-mirror to Drive.
