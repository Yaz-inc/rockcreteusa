Scope

What's in, what's out.

Explicit list of contracted scope. Anything not on this page is handled via a written change order.

In scope

  • Site rebuild on WordPress + WooCommerce, desktop and mobile responsive.
  • Catalog migration: 300 active products + 300+ discontinued products (as Draft under "Discontinued Products").
  • Migration of 1,246 customer accounts and 7,975 historical orders, with customer↔order↔product relationships preserved.
  • Core integrations: Stripe, PayPal, Avalara (tax), shipping zones, label automation and tracking, GA4 Enhanced eCommerce.
  • Branded transactional emails: order confirmation, shipping, cancellation, refund.
  • Performance targets: <3 second load, 80+ Desktop PageSpeed (depends on hosting and third-party scripts). See Performance Targets.
  • Migration protections: pre-migration backup, validation checklist, written sign-off, 7-day rollback window.
  • Client team training and handoff documentation in Phase 4.
  • Warranty: critical bugs resolved within 24 hours; non-critical fixes within the 30-day post-launch window.
  • Plugin budget cap: up to $2,500/year in plugin licenses/subscriptions, paid directly by Rockcrete; anything above requires prior approval.
  • Annual plugin / license budget cap (see New Minds Group for the dollar figure).

Discovery & Architecture

What happens in Phase 1

This phase sets the base for the new site before visual design starts.

  • We start with the kickoff meeting and review the full project brief.
  • We audit the current site: content, SEO, performance, and plugins.
  • We identify discontinued products that still matter for old orders.
  • We create the site structure and page organization plan.
  • We create UX wireframes for the key shopping pages.
  • We map the navigation and main user flows.
  • We set up the staging environment and the base WordPress + WooCommerce install.
  • We audit the plugin setup and prepare a plugin shortlist for approval.
  • We prepare the plan for moving products, pages, and images to the new site.
  • We close the phase with a Phase 1 sign-off meeting.

Discontinued products

Products no longer sold but referenced in historical orders are migrated as Draft status under a dedicated "Discontinued Products" category. They are not searchable or browsable on the storefront, but are visible when a customer or admin views a historical order; they are not purchasable. This preserves order line item integrity and admin records without cluttering the storefront.

Identified through a dedicated audit in Phase 1. Estimated at 300+ discontinued products (totaling ~600+ in the migrated catalog).

Migration protections

Pre-migration

  • Full database dump (SQL) exported to off-site backup.
  • Full file system backup (wp-content, /uploads).
  • Backup integrity verified via isolated restore test.
  • Staging migration completed and validated before touching production.

Post-migration validation

  • Count reconciliation: products / customers / orders (staging = source).
  • Random sampling: 50+ records per type, manually spot-checked.
  • Order↔customer and order↔product relationships verified on sampled records.
  • Discontinued products confirmed as Draft under "Discontinued Products".
  • Images verified (no 404 / broken media links).
  • End-to-end checkout flow test completed on staging.

Client sign-off & DNS cutover

  • Written client sign-off received (email or signed document) before cutover proceeds.
  • Original site kept live on a staging/temp subdomain for the full 7-day window.
  • 7-day rollback window begins at the DNS cutover timestamp.
  • Client notified at day 5 that the rollback window expires in 48 hours.
  • Rollback window closure requires written confirmation from both parties.

Cómo se calculan estos números

Los datos provienen del transcripto JSONL que Claude Code mantiene por sesión local. El script telemetry/extract-telemetry.py recorre el archivo, agrega tokens reportados por la API por cada turno asistente, y aplica las tarifas vigentes de Sonnet 4.x para producir el costo. El tiempo activo se calcula sumando los intervalos entre turnos consecutivos cuando son menores a 30 minutos (intervalos mayores se asumen como pausas humanas).

El equivalente humano usa 10× como multiplicador del tiempo activo IA, punto medio del rango 8–15× que la industria reconoce para tareas de análisis, redacción técnica e investigación documental ejecutadas a nivel consultor sénior. Tarifa horaria base: $125 USD.

Standing rules

  • Send-by dates are internal targets, not legal obligations. The contract obligates Hank to pay by the due date column. The send-by column is the latest we should email the invoice to give him a comfortable window.
  • No late fee is in the contract (1.5% monthly clause removed by Daniel 2026-05-06). The lever for late payment is the Section 1 right to pause work until balances clear.
  • Holiday avoidance is silent — don't add explanatory notes about Juneteenth or Independence Day in the invoice itself.
  • Payment method — confirm with Hank during Phase 1 kickoff. ACH preferred; wire or check acceptable. Card surcharge applies if used (set policy with Daniel).

Cascade dependencies

  • Phase 2 invoice (Jun 15) assumes Phase 1 sign-off (Jun 16) lands on schedule. If Phase 1 slips, push the Phase 2 invoice send-date by the same number of business days (Section 2 cascade clause).
  • Phase 3 invoice (Jul 6) depends on Phase 2 Design Approval Sign-Off (Jul 7). Same cascade rule.
  • Phase 4 invoice (Sep 30) depends on Phase 3 sign-off (Sep 29). If go-live slips, hold the invoice until sign-off is confirmed in writing.
